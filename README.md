# Climate Resilience Fund — Evidence Chatbot

A **local RAG (Retrieval-Augmented Generation) chatbot** that lets users ask natural-language questions about climate project field notes and receive answers with verifiable citations.

Built to answer the question: *"Can we trust narrative justifications, and can a reviewer verify claims?"*

![Python](https://img.shields.io/badge/Python-3.10+-blue) ![Flask](https://img.shields.io/badge/Flask-lightweight-green) ![No API Key](https://img.shields.io/badge/API_Key-not_required-brightgreen)

## Quick Start

```bash
# 1. Install Flask (only dependency)
pip install flask

# 2. Run
python chatbot_ui.py

# 3. Browser opens automatically to http://127.0.0.1:5000
```

No API key. No internet (after install). No database.

## How It Works

The chatbot uses a **keyword-scored retrieval pipeline** over 90 structured field notes:

### 1. Retrieve
When you ask a question, every field note is scored:
| Signal | Points |
|---|---|
| Exact file name mentioned (e.g. `field_note_028.txt`) | +100 |
| District code match (e.g. `D-010`) | +50 |
| Each keyword found in note text | +1 |
| Keyword matches risk category | +3 |

The **top 5** scoring notes are selected.

### 2. Answer
Each selected note is parsed with regex to extract structured fields (partner, severity, observations, mitigations). Every piece of information gets a citation:

```
[field_note_009.txt: "Partner: Horizon Resilience Group"]
```

### 3. Display
Answers appear as cards in a chat UI, with blue citation badges linking every claim back to its source file and quoted text.

## Design Decisions

- **No LLM dependency** — answers are generated by parsing structured notes, not by calling an API. This means zero cost, zero latency, and fully offline operation.
- **Citation-first** — every claim in the response includes `[filename: "quoted text"]` so a reviewer can verify it against the source document.
- **Single-file app** — the HTML, CSS, JS, and Python backend are all in `chatbot_ui.py` for maximum portability.
- **Keyword scoring > embeddings** — for structured, consistently-formatted documents, a weighted keyword scorer with domain-specific boosts (district codes, risk categories) outperforms generic embedding similarity.
